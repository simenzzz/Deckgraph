# Integration Schemas

> **Canonical source:** `packages/shared/src/types/integration.ts`
> **Last verified:** Pre-implementation (schema designed, not yet coded)

Types for integrations, templates, and integration results.

## Integration

Represents a third-party service integration detected in or available for the project.

```typescript
interface Integration {
  /** Unique identifier, e.g. "stripe", "clerk", "supabase" */
  id: string
  /** Display name */
  name: string
  /** Short description */
  description: string
  /** Category for marketplace grouping */
  category: IntegrationCategory
  /** URL to the service's logo/icon */
  iconUrl: string
  /** Official documentation URL */
  docsUrl: string
  /** Current status in this project */
  status: IntegrationStatus
}

type IntegrationCategory =
  | "payments"
  | "auth"
  | "database"
  | "storage"
  | "email"
  | "analytics"
  | "ai-ml"
  | "maps"
  | "search"
  | "monitoring"
  | "messaging"

type IntegrationStatus =
  | "not_installed"
  | "installing"
  | "installed"
  | "update_available"
  | "error"
  | "removing"
```

## IntegrationTemplate

A curated template that defines how to integrate a service into a specific framework.

```typescript
interface IntegrationTemplate {
  /** Kebab-case ID, e.g. "stripe-nextjs", "clerk-express" */
  id: string
  /** Service this template integrates */
  service: string
  /** Framework this template targets */
  framework: string
  /** Human-readable description */
  description: string
  /** Template version */
  version: string
  /** Required configuration keys (user must provide values) */
  requiredConfig: readonly ConfigField[]
  /** Files that will be created or modified */
  files: readonly TemplateFile[]
  /** Dependencies to install */
  dependencies: readonly TemplateDependency[]
}

interface ConfigField {
  /** Key name, e.g. "STRIPE_SECRET_KEY" */
  key: string
  /** Human-readable label */
  label: string
  /** Help text explaining where to find this value */
  helpText: string
  /** Whether this field is required */
  required: boolean
  /** Placeholder text */
  placeholder?: string
}

interface TemplateFile {
  /** Relative path from project root */
  path: string
  /** What this file does (shown to user) */
  purpose: string
  /** "create" for new files, "modify" for patches to existing files */
  operation: "create" | "modify"
  /** Template content (with {{ config.KEY }} placeholders) */
  content: string
}

interface TemplateDependency {
  /** Package name */
  name: string
  /** Version range */
  version: string
  /** Whether this is a dev dependency */
  isDev: boolean
}
```

## IntegrationResult

Generated by the analysis pipeline after scanning the project and composing a prompt. Contains both directly executable safe actions and a structured prompt for complex changes.

```typescript
interface IntegrationResult {
  /** Unique result ID */
  id: string
  /** Service being integrated */
  service: string
  /** Template used (if any) */
  templateId: string | null
  /** Plain-language explanation of what was analyzed and planned */
  explanation: string
  /** Safe actions that are auto-executed (npm install, .env, new files) */
  directActions: readonly PlannedAction[]
  /** Structured prompt for complex changes (null if all actions are safe) */
  prompt: StructuredPrompt | null
  /** Project files bundled for prompt context (null if no prompt needed) */
  contextBundle: ContextBundle | null
  /** Timestamp */
  createdAt: string
}

interface PlannedAction {
  /** Unique action ID */
  id: string
  /** Action type */
  type: "install" | "create_file" | "modify_file" | "delete_file" | "config" | "uninstall"
  /** Plain-language description of what this action does */
  description: string
  /** Detailed info (file path, package name, diff preview, etc.) */
  details: Record<string, unknown>
  /** Safety classification */
  safety: "safe" | "complex"
}
```

See [Prompt Schema](./prompt.md) for `StructuredPrompt` and `ContextBundle` type definitions.

## Template Directory Structure

Templates are stored in the MCP server's `templates/` directory, organized by category:

```
templates/
├── payments/
│   └── stripe/
│       ├── nextjs.json
│       ├── express.json
│       └── ...
├── auth/
│   ├── auth0/
│   ├── clerk/
│   └── firebase-auth/
├── database/
│   ├── supabase/
│   ├── prisma/
│   └── mongodb/
└── storage/
    ├── s3/
    └── cloudinary/
```

Template IDs use kebab-case: `stripe-nextjs`, `clerk-express`, `supabase-nextjs`.
